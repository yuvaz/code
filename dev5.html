if the Switcher is secondary part of the header which means placed at rignt h1 should be replaced by h2 tag, as h1 can be only one on HTML page for accesibility rules.



switcher - This class is the parent of all complete Account switcher, this can be placed in the IA header either lef or right.

switcherTrigger - This id holds click event which triggers the switcher to open.

switcher_header - This class holds the content of header part, title and subtitles.

module_title - This class renders the styles of the Module title, this can be used when you want to show as title if not you can remove this class which will look like general text on the header.

switcher_content - parent of the Switcher content container, this will also be used in functionality to show on click of switcherTrigger.

switcher_list - parent for every listing, contain the border, padding and cursor

switcher_list_group - This class serves as group wrapper and childs the each list items, if any module needs to iterate groups or grouping is required this group has to be used.

switcher_group_header - This class should be used with H3 tag and makes the group heading.

switcher_list_item - This class represents the each list item, on this level you can iterate the list as much as you can.

switcher_column - This class is the parent of the two column structure with the combination of Switcher6040

simple6040 - table structure left is 60% and right is 40%

switcher_title - this class serves the title of the each list item.

lnk_primary - generic class used to inject the branding color and size

switcher_subtitle - This class wrapps the subtitles of the switcher header.

Functionality

commons.validations.accountSwitcher();

Note:commons file is mandatory required to make it work.

         To make the list item selected "selected" class need to be added to the respective list item.

         For extra requirements of the applications should be handled by respective application only.
